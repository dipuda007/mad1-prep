What is Git? What is GitHub? (viva staple)

Git is a version control system (VCS): it records snapshots ("commits") of your project so you can see history and go back. It is distributed — every machine has the complete history, no central server required to work. GitHub is a website that hosts git repositories online — your backup + the place IITM tracks your milestones. Git = tool, GitHub = hosting service.

Logging OUT an old GitHub account (Windows)

Windows stores GitHub logins in Credential Manager, not in git itself. If this PC ever pushed as a different GitHub user, remove that first:

Way 1 — command line:

cmdkey /list | findstr github

If you see entries like LegacyGeneric:target=git:https://github.com, delete:

cmdkey /delete:LegacyGeneric:target=git:https://github.com

Way 2 — GUI (easier): press Win, type Credential Manager → Windows Credentials → find every entry mentioning github.com → Remove.

Way 3 — via git's credential manager:

git credential-manager github logout

Any one of the three works. After this, the machine has no GitHub login.

Daily workflow — the only 4 commands you need

git status                     # what changed? (red = unstaged, green = staged)
git add .                      # stage ALL changes in this folder
git commit -m "message"        # snapshot with a message
git push                       # upload to GitHub

That's it. After each milestone: add → commit with the milestone's unique message → push. Any number of extra commits is allowed — the milestone just needs one commit with its unique message.

Connecting your project folder to your existing repo

Files git should ignore

Create a file named exactly .gitignore in the project folder:

__pycache__/
instance/
venv/
*.pyc

The database and Python cache are machine-generated — they don't belong in history. (Viva: "what is .gitignore?" — a list of path patterns git must not track.)

"Is git centralized or distributed?" — distributed: every clone holds the full history; you can commit offline. Centralized VCS (like old SVN) needs the central server for every operation. "Git vs GitHub?" — tool vs hosting. "Uses of git besides tracking?" — collaboration, branches for experiments, reverting mistakes, and here: proving the project is your own work over time (IITM checks your commit history as plagiarism evidence!).
