Lesson 3: Python II — lists, dicts, if, loops

Goal: the four structures your whole app is made of.

Lists — ordered collections

events = ["Music Night", "Valley of Flowers", "Kedarkantha"]

print(events[0])        # Music Night   (counting starts at 0!)
print(len(events))      # 3
events.append("Roopkund")   # add to the end
print(events)

In your app, Event.query.all() gives you a list of event objects — that's why lists matter.

"Difference between list and tuple?" — asked in almost every viva. A list [1,2,3] is mutable (can change: append, remove, reassign). A tuple (1,2,3) is immutable (cannot change after creation). Tuples are slightly faster and can be used as dictionary keys. "Mutable" = changeable in place.

Dictionaries — key → value pairs

event = {"name": "Music Night", "slots": 10, "category": "Hard"}

print(event["name"])       # Music Night
event["slots"] = 9         # change a value
event["location"] = "Nepal"   # add a new key

You use dicts in the app: request.form['username'] works exactly like a dictionary, and the admin dashboard passes a stats dict to the template.

if / elif / else

slots = 0

if slots <= 0:
    print("No slots left")
elif slots < 3:
    print("Filling fast!")
else:
    print("Plenty of space")
Indentation is grammar in Python. The 4 spaces before print tell Python it belongs to the if. Wrong indentation = error or wrong logic. VS Code indents for you when you type : and Enter.

for loops

events = ["Music Night", "Valley of Flowers", "Kedarkantha"]

for t in events:
    print(f"Available: {t}")

# loop over numbers
for i in range(3):     # 0, 1, 2
    print(i)

The same loop appears in Jinja templates as {% for event in events %} to build the event tables — one HTML row per event.

Combining them — mini booking logic

bookings = []          # empty list
event = {"name": "Music Night", "slots": 2, "status": "Open"}

user = "ravi"
if event["status"] != "Open":
    print("Event not open")
elif event["slots"] <= 0:
    print("No slots left")
elif user in bookings:
    print("Already booked")
else:
    bookings.append(user)
    event["slots"] -= 1
    print(f"Booked! {event['slots']} slots left")

Read that again — it IS your project's booking route, just without the database. If you understand this block, you understand the heart of the app.
