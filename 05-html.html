Lesson 5: HTML — the skeleton of every page

Goal: read/write basic HTML. Make a file test.html anywhere, double-click it to open in your browser. No server needed for plain HTML.

A complete minimal page

<!DOCTYPE html>
<html>
<head>
    <title>My first page</title>
</head>
<body>
    <h1>Event-booking Management</h1>
    <p>Welcome to my app.</p>
</body>
</html>
  • <!DOCTYPE html> — tells the browser "this is HTML5".
  • <head> — invisible metadata: title (browser tab text), CSS links.
  • <body> — everything visible.
  • Tags open <p> and close </p>.

The tags your project uses

TagMeaning
<h1>…<h6>headings, big → small
<p>paragraph
<a href="/login">Login</a>link
<div>generic box for grouping (block)
<span>generic inline piece of text
<table> <tr> <th> <td>table → row → header cell → data cell
<form> <input> <select> <button>forms (next section)
<img src="pic.png">image (no closing tag needed)
"Difference between block and inline elements?" — a block element (div, p, h1, table, form) starts on a new line and takes the full width. An inline element (span, a, strong, img, input) flows inside the line and only takes the width of its content.

Tables — you will write many

<table>

        <th>Event</th>
        <th>Slots</th>
    </tr>

        <td>Music Night</td>

Forms — how users send data

<form method="POST" action="/login">
    <label>Username</label>
    <input type="text" name="username" required>

    <label>Password</label>
    <input type="password" name="password" required>

    <button type="submit">Login</button>
</form>
  • method="POST" — send data hidden in the request body (right for passwords / changes).
  • action="/login" — which URL receives the data.
  • name="username" — the key Python uses: request.form['username']. No name attribute → Flask receives nothing. #1 beginner bug.
  • required — browser refuses to submit if empty. This is HTML5 front-end validation.
"What is name vs id in HTML?" — name is the key under which a form field's value is sent to the server. id is a unique label for the element inside the page, used by CSS (#id) or JavaScript; it is never sent with the form.

Other input types you'll use: type="number", type="date", and dropdowns:

<select name="category">
    <option value="Easy">Easy</option>
    <option value="Hard">Hard</option>
</select>
