Lesson 0: Setup

Goal: Python + VS Code working, one project folder, one command run.

1. Install Python

You already have Python 3.14 installed (this prep confirmed it). To check on any machine, open PowerShell (press Win, type powershell, Enter) and run:

python --version

You should see something like Python 3.14.5. If instead Windows opens the Microsoft Store, install Python from python.org and during install tick "Add python.exe to PATH".

2. Install VS Code

Download from code.visualstudio.com. After installing, open it and install one extension: press Ctrl+Shift+X, search Python (by Microsoft), click Install.

3. Make YOUR project folder

This will be the folder you build the EventHub demo app in (you will make a second, separate folder later for your own project). In PowerShell:

cd C:\projects
mkdir eventhub-app
cd eventhub-app
code .

code . opens the current folder in VS Code. From now on you work here.

4. The VS Code terminal

Inside VS Code press Ctrl+` (the key above Tab). A terminal opens at the bottom, already inside your folder. This is the same PowerShell — you will use this for everything: running Python, running Flask, git commands.

5. First run

In VS Code: File → New File → name it hello.py → type:

print("I am going to pass this viva")

Save with Ctrl+S. In the terminal:

python hello.py

If the sentence prints, your machine is ready.

6. Install the two packages the project needs

pip install flask flask-sqlalchemy

If pip alone is not recognized, use python -m pip install flask flask-sqlalchemy.

Virtual environments (venv): tutorials often start with venv. The official viva checklist itself says: "Skip virtual environments if they cause issues, install packages globally if needed." We install globally — one less thing to break during the viva. If asked in viva: a venv is an isolated folder of packages per project so different projects can use different package versions; created with python -m venv venv, activated with venv\Scripts\activate.
"Which frameworks did you use?" — Flask (backend web framework), Jinja2 (template engine, comes with Flask), Flask-SQLAlchemy (ORM for the SQLite database), Bootstrap (CSS framework for the front end). These are exactly the ones the project statement mandates.
