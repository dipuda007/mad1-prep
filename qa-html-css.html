Q&A: HTML, CSS & Bootstrap (36 questions)

HTML

1. What is HTML?

HyperText Markup Language — defines the structure/content of web pages using tags. Not a programming language (no logic), a markup language.

2. What does <!DOCTYPE html> do?

Declares the document as HTML5 so browsers render in standards mode instead of legacy "quirks mode".

3. What goes in <head> vs <body>?

head: invisible metadata — title, charset, CSS links, viewport. body: everything rendered on screen.

4. What is metadata in HTML? Where is it?

Data about the page, in <meta> tags inside head: charset="UTF-8" (text encoding), viewport (mobile scaling), description, etc.

5. Block vs inline elements?

Block (div, p, h1, form, table): new line, full width. Inline (span, a, img, input, strong): flows within the line, content width only.

6. div vs span?

Both are meaningless generic containers for styling/grouping — div is block, span is inline.

7. name vs id attribute?

name = the key used when form data is sent to the server (request.form['name']). id = unique page-internal identifier for CSS/JS targeting; not submitted with forms.

8. What is the action attribute of a form? And method?

action = URL the data is sent to (defaults to the current URL if omitted). method = GET or POST — HTML forms support only these two verbs.

9. Input types you used?

text, password (masked), number (with min=), date (gives YYYY-MM-DD), plus select dropdowns and submit buttons.

10. What is the required attribute?

HTML5 frontend validation — browser blocks submitting an empty field. Convenience only; my backend still validates because requests can bypass the browser.

11. How do you make a link? A table?

<a href="/login">Login</a>. Table: <table> containing <tr> rows, with <th> header cells and <td> data cells.

12. Cell padding vs cell spacing? (asked constantly)

Cell padding = space INSIDE a cell, between its content and its border. Cell spacing = space BETWEEN neighbouring cells' borders. In modern CSS: padding on td vs border-spacing on table.

13. How to add an image? Background image?

<img src="{{ url_for('static', filename='pic.png') }}">. Background: CSS body { background-image: url('pic.jpg'); background-size: cover; }. Image + gradient: background-image: linear-gradient(rgba(0,0,0,.5), rgba(0,0,0,.5)), url('pic.jpg');

14. What is the DOM? Which language manipulates it?

Document Object Model — the browser's in-memory tree of the page's elements. JavaScript manipulates it (change text, hide elements) without reloading. My core app doesn't use JS, so pages update via fresh server-rendered HTML instead.

15. Static vs dynamic pages?

Static: same fixed HTML for everyone (this prep app!). Dynamic: HTML generated per request with data — every page of my project (Jinja fills in events/bookings from the DB).

16. Can HTML alone send HTTP requests?

Yes — links send GET, forms send GET or POST, and the browser requests images/CSS automatically. For PUT/DELETE you need JS or a tool like Postman.

CSS

17. What is CSS? Why "cascading"?

Cascading Style Sheets — rules for how elements look. "Cascading" = when rules conflict, they resolve in a defined order: importance, specificity, then source order.

18. Three ways to include CSS + priority order?

Inline (style="..." on the tag) — highest priority; Internal (<style> block in head); External (.css file via <link>). Internal vs external have equal footing — specificity then last-loaded wins. My project: external (Bootstrap CDN + my static/style.css).

19. Why does inline CSS beat the others?

Highest specificity — it's attached directly to the element, so it overrides id, class and tag rules (only !important beats it).

20. Selector types?

Tag h1 {}, class .card {} (many elements), id #logo {} (one element). Specificity: id > class > tag.

21. How to center text / an element?

Text: text-align: center; (Bootstrap: class text-center). Block element: margin: 0 auto; with a width (Bootstrap: mx-auto), or flexbox display:flex; justify-content:center; (Bootstrap: d-flex justify-content-center).

22. How to change background/font colour?

body { background-color: #f0f5f1; color: #222; } — my style.css does exactly the background one. Live task favourite: find the element, add the rule in static/style.css, hard-refresh (Ctrl+F5).

23. Padding vs margin?

Padding = space inside the border (between border and content). Margin = space outside the border (between this element and neighbours). Box model order: content → padding → border → margin.

24. What is the CSS box model?

Every element is a rectangle of content wrapped by padding, then border, then margin. Explains all spacing questions.

25. What is !important?

Suffix forcing a rule to win over normal specificity: color: red !important;. Last resort — breaks the natural cascade.

26. What is a pseudo-class like :hover?

Styles for a state: a:hover { color: green; } applies while the mouse is over. Bootstrap's table-hover uses this.

Bootstrap

27. What is Bootstrap? Why use it?

A CSS framework of ready-made classes (buttons, cards, grid, navbar). Fast, consistent, responsive out of the box; the only CSS framework the project permits.

28. How did you include Bootstrap?

One <link> tag in base.html loading bootstrap.min.css from the jsDelivr CDN — inherited by all pages through template inheritance.

29. What is a CDN?

Content Delivery Network — distributed servers hosting common files close to users. Browser downloads Bootstrap from the CDN instead of my server. Needs internet; offline alternative is downloading the css into static/.

30. How does the Bootstrap grid work?

A row is divided into 12 columns. col-md-3 = span 3 of 12 (quarter width) on medium screens and up, stacking to full width below — that's what makes my dashboard responsive.

31. What does "responsive" mean?

Layout adapts to screen size (phone/tablet/desktop). Achieved via the viewport meta tag + Bootstrap's grid breakpoints (media queries under the hood).

32. What is a class in Bootstrap, e.g. btn btn-success?

Pre-written CSS classes: btn gives button shape/padding; btn-success colours it green; btn-sm shrinks it. Multiple classes combine, space-separated.

33. Which Bootstrap components did you use?

Navbar, container, grid (row/col), cards, tables, buttons, form controls, alerts (flash messages), badges (status labels), spacing utilities (mt-4, mb-3…).

34. What is bootstrap.min.css — why "min"?

Minified: whitespace/names compressed for smaller download; identical rules, just unreadable.

35. Move an element to the right / navbar links to the end? (live-task classic)

Bootstrap: add ms-auto (margin-start auto pushes the element right — my navbar uses it). Raw CSS: margin-left: auto; in a flex container, or float: right.

36. Change button position from bottom to top of a list? (live-task classic)

It's just HTML order — cut the button's HTML and paste it above the table/list in the template. Restart not needed; refresh the page (templates reload in debug mode).
