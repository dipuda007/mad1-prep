1. The schema on paper (draw this in your report as the ER diagram)

User(id PK, username UNIQUE, password_hash, name, role, approved, blacklisted)
  │ 1-to-1 → StaffProfile(id PK, user_id FK, phone, experience)
  │ 1-to-many → Booking(id PK, user_id FK, event_id FK, booking_date, status, payment_status)
  │ 1-to-many (as staff) → Event(id PK, name, location, category, duration_hours,
  │                            available_slots, status, start_date, end_date, staff_id FK)
Event 1-to-many → Booking

Design decisions you must be able to defend:

  • One User table for all three roles with a role column — simpler than three tables, and login is one lookup.
  • approved — staff register but can't log in until admin flips this to True.
  • blacklisted — blocks login for anyone; admin can toggle.
  • StaffProfile separate — the milestone asks for it; demonstrates a one-to-one relationship.
  • available_slots lives on Event and is decremented on booking / incremented on cancel — overbooking check is simply slots <= 0.
  • Dates stored as 'YYYY-MM-DD' strings — HTML date inputs give exactly that, and SQLite has no real date type anyway.

2. models.py — type it

# models.py -- all database tables live here (the "M" in MVC)

from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy()


class User(db.Model):
    # one table for all three roles: admin / staff / user
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(50), unique=True, nullable=False)
    password_hash = db.Column(db.String(200), nullable=False)
    name = db.Column(db.String(100), nullable=False)
    role = db.Column(db.String(20), nullable=False, default='user')
    approved = db.Column(db.Boolean, nullable=False, default=True)      # staff start as False
    blacklisted = db.Column(db.Boolean, nullable=False, default=False)

    # relationships (these are not columns, they are shortcuts to related rows)
    staff_profile = db.relationship('StaffProfile', backref='user', uselist=False)
    bookings = db.relationship('Booking', backref='user')
    assigned_events = db.relationship('Event', backref='staff')


class StaffProfile(db.Model):
    # extra details only staff members have (one-to-one with User)
    id = db.Column(db.Integer, primary_key=True)
    user_id = db.Column(db.Integer, db.ForeignKey('user.id'), nullable=False)
    phone = db.Column(db.String(15))
    experience = db.Column(db.String(200))


class Event(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    location = db.Column(db.String(100), nullable=False)
    category = db.Column(db.String(20), nullable=False)                # Music / Sports / Tech
    duration_hours = db.Column(db.Integer, nullable=False)
    available_slots = db.Column(db.Integer, nullable=False)
    status = db.Column(db.String(20), nullable=False, default='Pending')  # Pending/Approved/Open/Closed/Completed
    start_date = db.Column(db.String(10))    # stored as 'YYYY-MM-DD' text (SQLite has no date type)
    end_date = db.Column(db.String(10))
    staff_id = db.Column(db.Integer, db.ForeignKey('user.id'))          # which staff runs this event

    bookings = db.relationship('Booking', backref='event')


class Booking(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    user_id = db.Column(db.Integer, db.ForeignKey('user.id'), nullable=False)
    event_id = db.Column(db.Integer, db.ForeignKey('event.id'), nullable=False)
    booking_date = db.Column(db.String(10), nullable=False)
    status = db.Column(db.String(20), nullable=False, default='Booked')  # Booked / Cancelled / Completed
    payment_status = db.Column(db.String(20), nullable=False, default='Paid')

3. app.py — skeleton + admin created by code

# app.py -- Flask setup + all routes (the "C" in MVC). Templates are the "V".

from datetime import datetime

from flask import Flask, render_template, request, redirect, url_for, session, flash
from werkzeug.security import generate_password_hash, check_password_hash

from models import db, User, StaffProfile, Event, Booking

app = Flask(__name__)
app.config['SECRET_KEY'] = 'event-app-secret-key'                    # needed for session + flash
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///eventhub.db'     # file lands in instance/ folder
db.init_app(app)


# ---------- create tables + default admin, then start ----------

def setup_database():
    with app.app_context():
        db.create_all()
        if not User.query.filter_by(role='admin').first():
            admin = User(username='admin',
                         password_hash=generate_password_hash('admin123'),
                         name='Administrator',
                         role='admin')
            db.session.add(admin)
            db.session.commit()


if __name__ == '__main__':
    setup_database()
    app.run(debug=True)

Explanations you must own:

  • with app.app_context(): — database operations need to know which app's config to use; outside a request you must open the context manually. (Viva: "application context vs request context" — app context = app-level info like config/db, request context = info about one request like request.form; Flask opens both automatically during a request.)
  • db.create_all() — reads every db.Model class and CREATEs missing tables. This is the "database created programmatically" requirement — never touch DB Browser to create it.
  • The admin if — runs every start, inserts admin only if absent. This is the "Admin pre-created programmatically" requirement.
