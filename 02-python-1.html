Lesson 2: Python I — variables, strings, numbers

Goal: read and write basic Python lines. Make a file practice.py and type everything below into it, run with python practice.py.

Variables

A variable is a name pointing to a value. No type declarations needed — Python figures it out.

event_name = "Indie Music Night"   # text  -> str
slots = 10                        # whole number -> int
price = 4999.50                   # decimal -> float
is_open = True                    # True/False -> bool

print(event_name)
print(slots)

Strings (text)

name = "Ravi"
greeting = "Hello " + name        # joining strings with +
print(greeting)                   # Hello Ravi

# f-string: put variables inside text (used everywhere)
print(f"Event {event_name} has {slots} slots")

print(len(name))                  # 4  (length)
print(name.lower())               # ravi
print(name.upper())               # RAVI

Numbers

slots = 10
slots = slots - 1     # booking an event reduces slots -> 9
slots -= 1            # same thing, shorter -> 8
print(slots)

days = "5"            # careful: this is TEXT not a number
days = int("5")       # int() converts text to number — form inputs arrive as text!
print(days + 1)       # 6
"Why did you write int(request.form['duration_hours'])?" — everything that comes from an HTML form is a string; int() converts it to a number so the database stores a proper integer and comparisons like slots <= 0 work.

Comments

# anything after a hash is ignored by Python — notes for humans

Comparing things

print(slots == 0)    # equal?          False
print(slots != 0)    # not equal?      True
print(slots > 5)     # greater than?   True
# one = assigns, two == compares. Mixing them up is the classic beginner bug.
