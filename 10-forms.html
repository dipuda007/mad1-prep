Lesson 10: Forms — data from browser to Python

Goal: the full round trip. "Make a page with an input and a submit button that shows the text on another page" is one of the MOST repeated live viva tasks — this lesson IS that task.

The pattern (memorize)

# app.py
from flask import Flask, render_template, request, redirect, url_for

app = Flask(__name__)

@app.route('/ask', methods=['GET', 'POST'])
def ask():
    if request.method == 'POST':                # form was submitted
        name = request.form['username']         # read by the input's name attribute
        return redirect(url_for('show', person=name))
    return render_template('ask.html')          # first visit: just show the form

@app.route('/show/<person>')
def show(person):
    return f'Hello {person}!'
<!-- templates/ask.html -->
<form method="POST">
    <input type="text" name="username">
    <button type="submit">Submit</button>
</form>

What happens, step by step

  1. Browser GETs /ask → request.method is 'GET' → form page rendered.
  2. User types, clicks Submit → browser POSTs to the same URL with the data in the body.
  3. request.method is now 'POST' → we read request.form['username'].
  4. We redirect (302) → browser makes a fresh GET to /show/name.
You wantUse
data from a POST formrequest.form['field_name']
data from the URL query string (?q=x, GET form)request.args.get('q', '')
which method was usedrequest.method
"What if you remove method="POST" from the form?" — forms default to GET: the data moves into the URL as ?username=..., request.form becomes empty (you'd need request.args), and if the route only allows POST-handling logic the submit appears to do nothing.

"Why methods=['GET','POST'] in the route?" — routes accept only GET by default; POSTing to a GET-only route gives 405 Method Not Allowed.
"Why redirect after POST instead of returning the page directly?" — the Post/Redirect/Get pattern: if you return HTML straight from the POST, refreshing the page re-submits the form (double booking!). Redirecting makes the browser land on a normal GET page.

Search forms use GET (your project does this)

<form method="GET" action="/user">
    <input type="text" name="location">
    <button type="submit">Filter</button>
</form>
location = request.args.get('location', '')   # '' default if absent

Search doesn't change data, results should be shareable/bookmarkable → GET is correct. Login changes state and carries a password → POST is correct. Being able to SAY this sentence is viva gold.

Try it (the classic viva task)

Two-number adder: form with two type="number" inputs → submit → new page shows the sum. You need: int(request.form['a']) + int(request.form['b']). Build it without looking, 10 minutes max.
