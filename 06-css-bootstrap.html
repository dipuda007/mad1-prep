Lesson 6: CSS & Bootstrap

Goal: understand the 3 ways to style, then let Bootstrap do 95% of the work.

CSS in one paragraph

CSS = rules saying which elements get which looks:

selector { property: value; }

h1      { color: green; }            /* all h1 tags */
.card   { border: 1px solid gray; }  /* all elements with class="card" */
#logo   { width: 100px; }            /* the one element with id="logo" */

The 3 ways to attach CSS (viva favourite!)

WayHowPriority
Inline<p style="color:red"> directly on the tagHighest
Internal<style>…</style> block inside <head>Whichever rule is more specific; tie → the one written later wins
Externalseparate .css file linked with
<link rel="stylesheet" href="style.css">
"Why does inline CSS win?" — it is closest to the element, so its specificity is highest; the browser resolves conflicts by specificity (inline > id > class > tag), and when equal, the last-loaded rule wins — that is the "cascading" in Cascading Style Sheets. Your project uses an external file (static/style.css) plus Bootstrap (also external).

Bootstrap — pre-written CSS classes

Bootstrap is a CSS framework: someone already wrote thousands of polished rules; you just put their class names on your tags. Add ONE line inside <head>:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
"What is a CDN?" — Content Delivery Network: a network of servers around the world that host common files (like bootstrap.min.css) so browsers download them fast from a nearby server, instead of your server sending them. The link above loads Bootstrap from a CDN.

The only Bootstrap classes your project uses

ClassEffect
containercentered page column with side margins
row / col-md-3grid: a row split into 12 units; col-md-3 = 3/12 = quarter width on medium+ screens (this is what makes it responsive)
card, card-body, card-titlewhite box with border — used for login box, dashboard stats
table table-bordered table-hoverstyled table with borders + hover highlight
btn btn-success btn-smbutton; success=green, danger=red, primary=blue; sm=small
form-control / form-select / form-labelstyled inputs, dropdowns, labels
navbar navbar-dark bg-successthe green top navigation bar
alert alert-infothe message boxes for flash messages
badge bg-secondarysmall gray label — used for statuses like "Open"
mb-3 mt-4 ms-auto w-100 d-inline text-centerspacing/layout utilities: margin-bottom, margin-top, margin-start auto (push right), full width, inline display, centered text

That's the entire list. If a proctor points at any class in your templates, the answer is in this table.

"Why Bootstrap?" — ready-made, tested, responsive (adapts to mobile/tablet via its grid), consistent look, no need to write hundreds of CSS lines. The project statement allows only Bootstrap as a CSS framework.

Your own CSS on top

Your project also has a small static/style.css — a page background colour, a green strip on stat cards, a soft card shadow. Written by you, easy to explain, proves you can write raw CSS too. Loaded after Bootstrap, so your rules win ties.

Try it

Take yesterday's test.html, add the Bootstrap link, wrap the content in <div class="container">, give the table class="table table-bordered" and the button class="btn btn-success". Refresh, admire.
