The zip — #1 cause of invalid submissions

Rules from the project document: a single zip; in the zip's root there is EXACTLY ONE folder (your project folder); everything lives inside it; at least one .py file present.

submission.zip
└── eventhub-app/          ← ONE folder in the root, nothing else
    ├── app.py
    ├── models.py
    ├── requirements.txt
    ├── templates/ ...
    └── static/ ...

Make it in PowerShell (from the folder ABOVE the project):

Compress-Archive -Path .\eventhub-app -DestinationPath .\submission.zip
Delete instance/, __pycache__/ and any venv folder before zipping. Do NOT zip "the files" — zip the FOLDER, so the zip root contains the folder, not loose files. After uploading, download your own submission back from the portal and open it to verify. Use the validation Google Form on the portal (unlimited tries) BEFORE the one-shot real submission.

The report (3–5 pages, PDF/DOCX, separate field)

  1. Student details (name, roll no 24f2008594, email).
  2. Project details: problem statement + how you approached it.
  3. AI/LLM declaration — mandatory even if 0%. State the percentage and what for. Honest + modest is safe; "assistance is allowed if you can explain and modify the code yourself".
  4. Frameworks/libraries used: Flask, Jinja2, Flask-SQLAlchemy, SQLite, Bootstrap.
  5. ER diagram — draw your 4 tables + relations (paper photo or draw.io; Build 1 has the text version).
  6. API endpoints — "None; core requirements implemented with server-rendered templates."
  7. Drive link of the video (set: anyone with the link can view!).

The video (3–10 min)

Script (matches the advised structure): 30s intro (who you are, what the app is) → 30s approach (Flask+SQLite, MVC, 4 tables) → 90s+ demo: the Build 6 end-to-end story (admin creates event → staff approval flow → user books → overbooking blocked → history) → 30s anything extra. Record with screen share; camera feed optional but recommended. Upload to your student Drive, access: anyone with the link.

Pre-viva checksum drill (do this once now, once before viva)

pip install checksumdir
# check.py  (place NEXT TO the unzipped folder, not inside)
import checksumdir
hash = checksumdir.dirhash("eventhub-app")
print("Directory Checksum:", hash)

At the viva you download your submission from the portal and either show the checksum matches or just run the downloaded copy directly. Practice: download → unzip → pip install -r requirements.txt → python app.py → running within 10 minutes. Rehearse until it's boring.
