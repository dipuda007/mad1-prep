1. Dashboard: my events + booking counts

@app.route('/staff')
def staff_dashboard():
    user = current_user()
    if not user or user.role != 'staff':
        return redirect(url_for('login'))

    events = Event.query.filter_by(staff_id=user.id).all()
    counts = {}
    for t in events:
        counts[t.id] = Booking.query.filter_by(event_id=t.id, status='Booked').count()
    return render_template('staff_dashboard.html', events=events, counts=counts)

Two things to be able to say: filter_by(staff_id=user.id) is what makes staff see only their events; the counts dict counts only status='Booked' (cancelled bookings shouldn't count toward "registered attendees").

2. Update slots + status

@app.route('/staff/event/update/<int:event_id>', methods=['POST'])
def staff_update_event(event_id):
    user = current_user()
    if not user or user.role != 'staff':
        return redirect(url_for('login'))

    event = Event.query.get_or_404(event_id)
    if event.staff_id != user.id:                   # only the assigned staff can touch this event
        flash('That event is not assigned to you.')
        return redirect(url_for('staff_dashboard'))

    event.available_slots = int(request.form['available_slots'])
    event.status = request.form['status']
    db.session.commit()
    flash('Event updated.')
    return redirect(url_for('staff_dashboard'))
The event.staff_id != user.id check is THE critical line. Without it any staff could edit any event by typing the URL. Role check = "is a staff member"; ownership check = "is THIS event's staff member". Two different protections — say both in the viva.

The status dropdown offers Approved / Open / Closed / Completed — changing to Completed is your "mark event as completed" requirement.

3. Participants list + manage bookings

@app.route('/staff/event/<int:event_id>/participants')
def participants(event_id):
    user = current_user()
    if not user or user.role not in ['staff', 'admin']:
        return redirect(url_for('login'))

    event = Event.query.get_or_404(event_id)
    if user.role == 'staff' and event.staff_id != user.id:
        flash('That event is not assigned to you.')
        return redirect(url_for('staff_dashboard'))

    bookings = Booking.query.filter_by(event_id=event.id).all()
    return render_template('participants.html', event=event, bookings=bookings)

Admin may also view this page (that's the ['staff', 'admin']), but only the assigned staff gets action buttons — the template checks {% if session.get('role') == 'staff' %}.

@app.route('/staff/booking/<int:booking_id>/<new_status>', methods=['POST'])
def staff_update_booking(booking_id, new_status):
    user = current_user()
    if not user or user.role != 'staff':
        return redirect(url_for('login'))

    booking = Booking.query.get_or_404(booking_id)
    if booking.event.staff_id != user.id:
        flash('That event is not assigned to you.')
        return redirect(url_for('staff_dashboard'))

    if new_status in ['Cancelled', 'Completed']:
        if new_status == 'Cancelled' and booking.status == 'Booked':
            booking.event.available_slots += 1      # give the slot back
        booking.status = new_status
        db.session.commit()
        flash('Booking marked ' + new_status + '.')
    return redirect(url_for('participants', event_id=booking.event_id))

Details worth noticing: booking.event.staff_id hops across the backref to check ownership; the in ['Cancelled','Completed'] whitelist stops URL tricksters sending status "Banana"; cancelling returns the slot.
