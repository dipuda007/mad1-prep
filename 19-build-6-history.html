1. Cancel booking (the last route)

@app.route('/user/cancel/<int:booking_id>', methods=['POST'])
def cancel_booking(booking_id):
    user = current_user()
    if not user or user.role != 'user':
        return redirect(url_for('login'))

    booking = Booking.query.get_or_404(booking_id)
    if booking.user_id != user.id:                 # you can only cancel your own booking
        return redirect(url_for('user_dashboard'))

    if booking.status == 'Booked':
        booking.status = 'Cancelled'
        booking.event.available_slots += 1
        db.session.commit()
        flash('Booking cancelled.')
    return redirect(url_for('user_dashboard'))

Same ownership pattern as staff: booking.user_id != user.id stops users cancelling other people's bookings by guessing IDs. Slot goes back. Booking row is NOT deleted — status becomes 'Cancelled' so history is preserved. That's the design answer for this milestone: we never delete booking history, we change status.

3. Status lifecycles (the two state machines)

Event:    Pending → Approved → Open → Closed → Completed
         (admin sets initial;  staff moves it once assigned;
          bookings allowed ONLY while Open)

Booking: Booked → Cancelled  (by user, or by staff)
         Booked → Completed  (by staff after the event)
