Lesson 4: Python III — functions, imports, classes

Goal: the last three Python ideas the project needs.

Functions — reusable blocks

def greet(name):
    return "Hello " + name

message = greet("Ravi")
print(message)            # Hello Ravi

def slots_left(total, booked):
    return total - booked

print(slots_left(10, 4))  # 6

Every Flask route is just a function. def login(): ... — that's it.

Imports — using other people's code

from datetime import datetime

now = datetime.now()
print(now.strftime('%Y-%m-%d'))    # e.g. 2026-07-07 — exactly how the app stamps bookings
# three shapes of import you'll see:
import math                          # use as math.ceil(2.1)
from flask import Flask, request     # pick specific names out of a package
from models import db, User          # import from YOUR OWN file models.py

Classes — blueprints for objects

A class describes what data a "thing" holds. Your database tables are classes.

class Event:
    def __init__(self, name, slots):
        self.name = name          # each event object carries its own name
        self.slots = slots

t1 = Event("Music Night", 10)          # t1 is an object (an "instance")
t2 = Event("Kedarkantha", 5)
print(t1.name, t2.slots)          # Music Night 5

In the real app you write class Event(db.Model): — the part in parentheses is inheritance: Event receives all abilities of SQLAlchemy's db.Model (saving to the database, querying) for free, and you only declare the columns.

"What is db.Model and why is it in the brackets?" — it is the base class from Flask-SQLAlchemy. Writing class Event(db.Model) means Event inherits from it, which is what turns a plain Python class into a database table with query powers (Event.query.all() etc.). This exact question appears constantly in vivas.

Decorators — the @ thing

@app.route('/login')
def login():
    ...

A decorator is a function that wraps another function to add behaviour without changing its code. @app.route('/login') tells Flask: "when a browser asks for /login, call this function". You don't need to write your own decorators — you only need to explain this one.

"What is if __name__ == '__main__':?" — Python sets the special variable __name__ to '__main__' only when the file is run directly (python app.py). If the file is imported by another file, the block is skipped. We use it so the server starts only when we run app.py itself.

Exceptions (bonus — sometimes asked)

try:
    days = int("abc")        # this crashes -> ValueError
except ValueError:
    print("Please enter a number")

try/except catches errors so the app shows a message instead of crashing.
