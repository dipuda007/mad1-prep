Q&A: Flask & Jinja2 (45 questions)

Flask basics

1. What is Flask? Advantages?

A lightweight ("micro") Python web framework mapping URLs to functions. Advantages: minimal and easy to learn, flexible (choose your own pieces), built-in dev server + debugger, Jinja2 included, huge extension ecosystem. Disadvantage: fewer batteries included than Django; you assemble more yourself.

2. Flask vs Django?

Flask: micro, unopinionated, pick your own ORM/auth. Django: full-featured (admin panel, ORM, auth built in), more structure, heavier. Flask suits small/medium apps and learning.

3. What does app = Flask(__name__) do?

Creates the application object. __name__ (the module's name) lets Flask locate the templates/ and static/ folders relative to the file.

4. What is @app.route and why is it needed?

A decorator registering a URL rule: "requests for this path call this function". Without it Flask has no idea which function serves which URL — the function would never run.

5. What is a view function / controller?

The function under @app.route — receives the request, runs logic (queries models), returns a response (usually render_template or redirect). In MVC terms these are my controllers.

6. Default port? Change it? Change host?

5000. app.run(port=8080). host='0.0.0.0' would accept connections from other machines on the network (default 127.0.0.1 = own machine only).

7. What is debug=True? Production?

Auto-restarts the server on code changes + interactive error pages with tracebacks. Must be OFF in production — the debugger can execute arbitrary code and leaks internals.

8. What can a Flask route return? Can it return a boolean?

String, rendered template, redirect, a dict/list (auto-converted to JSON), or a (body, status_code) tuple. A bare boolean raises TypeError — wrap it: str(True) or jsonify(ok=True).

9. What is a dynamic route?

URL with a variable part: @app.route('/event/<int:event_id>') — Flask converts the URL segment and passes it as the function argument. int: is a converter; without it you get a string.

10. What happens with two routes for the same URL?

First registered wins for matching; identical rule + endpoint raises an error. Also: one function can have multiple @app.route decorators stacked.

11. What is the request object?

Flask's per-request bundle of incoming data: request.method, request.form (POST body), request.args (URL query), request.files. Imported from flask; usable inside routes only.

12. request.form vs request.args?

form = data from a POST body (login, register, booking). args = data from the URL query string ?q=... (my search/filter forms which use GET).

13. Why methods=['GET','POST'] on some routes?

Routes accept only GET by default. Form pages need both: GET shows the empty form, POST processes the submitted data. POST to a GET-only route → 405 Method Not Allowed.

14. redirect vs render_template vs url_for?

render_template: fill a Jinja file with data, return HTML now. redirect: return a 302 telling the browser to request a different URL. url_for: build a URL string from a FUNCTION name (takes the endpoint name, e.g. url_for('login') → '/login'). Combo: redirect(url_for('dashboard')).

15. Why redirect after form POST?

Post/Redirect/Get pattern: refreshing after a direct POST response resubmits the form (duplicate booking). Redirect makes the browser land on a clean GET.

16. What is flash()? How is it shown?

Stores a one-time message in the session; base.html loops get_flashed_messages() into Bootstrap alerts; messages disappear once displayed. Requires SECRET_KEY because it rides on the signed session.

17. What is the SECRET_KEY? What breaks without it?

Key used to cryptographically sign the session cookie. Without it, using session or flash raises RuntimeError. Change it and all logged-in sessions become invalid.

18. What is the session? Where is it stored?

Per-visitor dict surviving across requests. Flask stores it INSIDE a signed cookie in the browser (client-side session) — readable if decoded, but not forgeable without the secret key. I keep user_id and role in it.

19. Cookie vs session?

Cookie: raw browser-side storage sent with every request. Session: the logical per-user data; Flask implements it via a signed cookie. Logout = session.clear().

20. How do users stay logged in "for so long" on real apps?

Persistent cookies with long expiry (session.permanent in Flask) or refresh tokens. My session cookie is a browser-session cookie — closes with the browser by default.

21. Application context vs request context?

App context = app-level things (config, db) active; request context = one specific request's data (request, session). Flask pushes both automatically per request; outside a request (my setup_database()) I push the app context manually: with app.app_context():.

22. What are Flask blueprints?

A way to split routes into modular groups (e.g. admin_bp, user_bp) registered onto the app — for organizing big projects. Mine is small enough that all routes live in app.py, deliberately.

23. What is Flask-Login / UserMixin? Why didn't you use it?

Extension automating sessions: login_user(), @login_required, current_user; UserMixin adds the methods it needs to your model. It's an optional milestone — I implemented sessions directly to show I understand the mechanism it abstracts.

24. What is WSGI?

Web Server Gateway Interface — the standard connecting Python apps to web servers. Flask is a WSGI app; the dev server speaks it; production uses gunicorn/waitress.

25. Status codes you see in the Flask terminal?

200 page served, 302 my redirects, 304 browser cache for static files, 404 wrong URL / get_or_404, 405 wrong method, 500 my code crashed.

26. How would you return JSON / build an API in Flask?

Return a dict — Flask jsonifies it: return {'name': event.name, 'slots': event.available_slots}. A REST API = routes like /api/events with GET/POST/PUT/DELETE methods returning JSON instead of HTML.

27. Difference between an API and a controller?

Both are routes; a controller returns rendered HTML for humans, an API endpoint returns data (JSON) for programs. Same Flask machinery, different response type.

28. Where do static files go? How are they linked?

static/ folder; linked with url_for('static', filename='style.css') — Flask serves them at /static/... automatically.

29. Why is your DB file inside an instance/ folder?

Flask-SQLAlchemy resolves relative sqlite:/// paths against Flask's instance folder — a place for files that belong to this installation and shouldn't be in version control. To relocate it I'd put an absolute path in the URI.

30. How does Flask handle routing internally (short)?

Each @app.route registers a URL rule in a map; on each request Werkzeug matches the path + method against the map and calls the matched endpoint function.

Jinja2

31. What is Jinja2?

Flask's template engine: HTML files with placeholders ({{ }}) and logic tags ({% %}) that Flask fills with Python data before sending.

32. {{ }} vs {% %} vs {# #}?

{{ expression }} outputs a value. {% statement %} is logic (if/for/extends/block). {# comment #} is a template comment, never rendered.

33. Which function renders templates? Where must they live?

render_template('file.html', name=value, ...) — extra kwargs become template variables. Files must be in the templates/ folder.

34. Write a Jinja for loop (live-task certainty).

{% for event in events %} <li>{{ event.name }}</li> {% endfor %} — endfor is mandatory.

35. Write a Jinja if/else.

{% if event.available_slots > 0 %} Book {% else %} Full {% endif %}

36. What is template inheritance? Show it.

base.html defines the shared skeleton with {% block %} holes; children start with {% extends 'base.html' %} and fill the blocks. All 12 of my templates extend base.html (navbar, Bootstrap link, flash messages written once).

37. What are Jinja filters?

Value transformers applied with |: {{ bookings|length }}, {{ name|upper }}, {{ price|round(2) }}. length is the one my templates use.

38. Can templates access the session?

Yes — Flask injects session (also request, url_for, get_flashed_messages). My navbar switches links based on session.get('role').

39. How does data get from a route into HTML?

Controller queries the DB → passes objects to render_template as keyword arguments → template prints them with {{ }} / loops them with {% for %} → finished HTML goes to the browser. (This is "how frontend connects to backend" — say it as this pipeline.)

40. What happens rendering an empty list into a table?

The for loop body runs zero times — headers render, no rows. My templates show a "none yet" paragraph via {% if not bookings %}.

41. How did you prefill the edit form?

value="{{ event.name if event else '' }}" — one event_form.html serves both add (event=None) and edit (real object); selects mark the current choice with {{ 'selected' if event and event.category == level }}.

42. Benefits of Jinja templating?

Separation of logic and presentation (V of MVC), no HTML built by string concatenation, automatic HTML escaping (protects against XSS injection), inheritance kills duplication.

43. What is autoescaping?

Jinja converts <, >, & in variables to safe entities, so a user named "<script>" can't inject running code into my pages — cross-site scripting (XSS) protection for free.

44. Loop with a condition: print list items longer than 3 chars.

{% for w in words %}{% if w|length > 3 %}{{ w }}{% endif %}{% endfor %}

45. Given [<Spot 1>,<Spot 2>…] with a flag, render conditionally (real past task).

Pattern answer: {% for s in spots %}{% if s.is_occupied %}<span class="badge bg-danger">{{ s.id }}</span>{% else %}<span class="badge bg-success">{{ s.id }}</span>{% endif %}{% endfor %} — loop + if + attribute access covers every variant they ask.
